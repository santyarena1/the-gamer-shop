PROMPT PARA CURSOR - NUEVO MÓDULO DE GENERACIÓN MASIVA DE IMÁGENES DE PC GAMER

Contexto general
================

Necesito crear un nuevo módulo dentro de mi sistema de gestión para generar imágenes publicitarias de PCs gamer en masa.

La idea es que el sistema tome una plantilla visual fija, el logo fijo de The Gamer Shop, una imagen variable del gabinete/PC y los componentes variables de cada equipo. Con esa información debe generar automáticamente una imagen cuadrada 1080x1080 lista para publicar.

El diseño de referencia es una pieza gráfica de The Gamer Shop con estética gamer premium: fondo negro, rojo intenso, blanco, tipografía grande, imagen de gabinete RGB a la derecha, componentes a la izquierda y una barra inferior con beneficios.

IMPORTANTE:
No quiero depender solamente de un prompt libre para una IA, porque eso puede generar errores de texto, cambios de diseño, deformaciones del gabinete o inconsistencias entre imágenes.
La recomendación principal es crear una plantilla programática y usar la IA solo como apoyo para mejorar la imagen del gabinete, integrar el fondo o generar variantes visuales controladas.


Objetivo del nuevo módulo
=========================

Crear un módulo llamado, por ejemplo:

- Generador de Imágenes
- Generador de Flyers
- Imágenes PC Gamer
- Plantillas de Publicación

El módulo debe permitir:

1. Cargar o seleccionar una PC/presupuesto/producto del sistema.
2. Tomar automáticamente los componentes principales.
3. Permitir editar manualmente los textos antes de generar.
4. Subir o seleccionar la imagen del gabinete/PC.
5. Generar una imagen final 1080x1080.
6. Descargar la imagen en PNG.
7. Guardar un historial de imágenes generadas.
8. Permitir regenerar una imagen usando los mismos datos.
9. Tener configuración de API para OpenAI desde una sección de configuración del sistema.


Recomendación técnica principal
===============================

La mejor solución para producción masiva es NO generar toda la imagen con IA desde cero.

El flujo recomendado es:

Sistema de gestión
    ↓
JSON con datos de la PC
    ↓
Template fijo HTML/SVG/Canvas
    ↓
Render automático a PNG
    ↓
Guardado en servidor / carpeta local / base de datos / storage
    ↓
Descarga o publicación

La IA debe usarse solo para tareas auxiliares:

1. Mejorar o limpiar la imagen del gabinete.
2. Eliminar fondo de la imagen del gabinete si hace falta.
3. Integrar mejor visualmente el gabinete con el fondo.
4. Generar fondos alternativos si se desea.
5. Nunca debe modificar los textos técnicos finales.

Los textos, posiciones, logo, iconos y estructura deben ser programáticos para evitar errores como:

- RYZEN escrito mal.
- GEFORCE escrito mal.
- BRONCE cambiado por BRONZE.
- GARANTÍA sin tilde si se quiere mantener con tilde.
- Datos técnicos inventados.
- Componentes alterados.
- Gabinete deformado.
- Logo deformado o mal ubicado.


Formato de salida
=================

El módulo debe generar imágenes en:

- Tamaño: 1080x1080 px.
- Formato: PNG.
- Calidad alta.
- Fondo oscuro.
- Sin precio, salvo que en el futuro se agregue una opción para mostrar precio.
- Preparado para Instagram, WhatsApp, web y publicaciones comerciales.

Configuración de API OpenAI
===========================

Agregar dentro del sistema una sección nueva en Configuración llamada:

Integración API

Dentro de esa sección agregar un bloque específico:

OpenAI

Campos necesarios:

1. API Key / Token de OpenAI
   - Campo tipo password.
   - Permitir mostrar/ocultar.
   - No mostrar completo luego de guardarlo.
   - Mostrarlo enmascarado, por ejemplo: sk-****abcd.

2. Modelo por defecto
   - Selector.
   - Ejemplos:
     - gpt-image-1
     - otro modelo disponible según implementación futura.

3. Estado de conexión
   - Botón: Probar conexión.
   - Debe hacer una llamada simple para verificar que el token funcione.
   - Mostrar estado:
     - Conectado correctamente.
     - Token inválido.
     - Error de conexión.
     - Sin configurar.

4. Guardar configuración
   - Botón para guardar cambios.

5. Eliminar token
   - Botón para borrar la API Key guardada.

Seguridad:

- No guardar el token en el frontend.
- Guardarlo del lado backend.
- Idealmente guardarlo cifrado o como variable segura.
- Nunca imprimir el token completo en consola.
- Nunca devolver el token completo al frontend.
- El frontend solo debe saber si existe token configurado y una versión enmascarada.
- No exponer el token en respuestas de API.

Endpoints sugeridos:

GET /api/settings/integrations/openai
Devuelve:
{
  "configured": true,
  "maskedKey": "sk-****abcd",
  "defaultModel": "gpt-image-1",
  "status": "connected"
}

POST /api/settings/integrations/openai
Recibe:
{
  "apiKey": "sk-xxxxxxxx",
  "defaultModel": "gpt-image-1"
}

POST /api/settings/integrations/openai/test
Prueba conexión con OpenAI y devuelve:
{
  "success": true,
  "message": "Conectado correctamente"
}

DELETE /api/settings/integrations/openai
Borra la configuración guardada.

Backend recomendado
===================

Para render programático:

Opción 1:
- HTML + CSS + Puppeteer/Playwright.
- Generar una página HTML con la plantilla.
- Sacar screenshot 1080x1080.
- Guardar como PNG.

Opción 2:
- SVG dinámico.
- Insertar textos, logo, iconos e imagen.
- Convertir SVG a PNG con Sharp.

Opción 3:
- Canvas server-side.
- Usar node-canvas o similar.
- Dibujar todo programáticamente.

Recomendación:
Usar HTML/CSS + Playwright o Puppeteer porque permite ajustar diseño más fácil y mantener una plantilla visual parecida a un flyer.


Flujo recomendado de generación
===============================

1. Usuario entra al módulo Generador de Imágenes.
2. Carga datos de una PC o los escribe manualmente.
3. Sube imagen del gabinete.
4. El sistema arma un JSON.
5. El frontend muestra preview.
6. Al tocar Generar:
   - Backend toma JSON.
   - Renderiza plantilla HTML/SVG.
   - Exporta PNG 1080x1080.
   - Guarda el archivo.
   - Devuelve URL del archivo generado.
7. Usuario puede descargar o guardar en historial.


Uso de OpenAI dentro del módulo
===============================

OpenAI debe ser opcional.

El sistema debe funcionar aunque no haya API Key configurada, usando render programático.

Si hay API Key configurada, se pueden habilitar acciones como:

1. Mejorar imagen de gabinete.
2. Quitar fondo.
3. Crear fondo gamer alternativo.
4. Integrar la PC al fondo.
5. Generar variantes visuales.

Pero no debe usarse IA para escribir los textos finales sobre la imagen, salvo que sea estrictamente necesario.

Regla importante:
Los textos finales deben venir siempre del JSON y renderizarse desde el sistema para evitar errores de IA.
